# Современный локальный музыкальный плеер

Стильный, высококачественный локальный музыкальный плеер для Android, разработанный полностью на Kotlin и Jetpack Compose. Приложение фокусируется на производительности, удобном доступе к локальным файлам и красивом пользовательском интерфейсе с плавными анимациями и современными паттернами проектирования.

## Ключевые возможности

### Плавающая навигация в стиле Telegram
Единая плавающая капсула в нижней части экрана, объединяющая активный мини-плеер и основные навигационные вкладки, обеспечивая чистый интерфейс без лишних границ.

### Динамическое и быстрое сканирование файлов
Использует Android Storage Access Framework (SAF) для безопасного доступа к выбранным пользователем каталогам. Оснащён оптимизированным рекурсивным механизмом сканирования, который кеширует метаданные файлов (размер, URI и дата последнего изменения) в локальную базу данных Room SQLite. Это обеспечивает молниеносную скорость при последующих сканированиях и перезапусках приложения, а файлы обновляются динамически, не исчезая из библиотеки.

### Интерфейс проигрывателя с винилом
Главный экран плеера украшает элегантная анимация вращающейся виниловой пластинки с отображением извлечённой обложки текущего трека.

### Стекломорфизм и оформление
Красивые полупрозрачные UI-компоненты, эффекты размытия и адаптивные цвета, которые корректно поддерживают как тёмную, так и светлую системную тему.

### Фоновое воспроизведение
Надёжное непрерывное воспроизведение аудио с эффективным управлением.

## Технологический стек и архитектура

- **Язык:** Kotlin
- **UI Фреймворк:** Jetpack Compose и Material Design 3
- **Локальное хранилище:** Room Database для безопасного и эффективного кеширования музыкальной библиотеки
- **Асинхронная обработка:** Kotlin Coroutines и Flow для реактивного управления состоянием и неблокирующего ввода-вывода
- **Архитектура:** MVVM (Model-View-ViewModel), обеспечивающая чёткое разделение ответственности между UI, состоянием плеера и получением данных

## Структура проекта

### `com.example.ui`
Содержит все файлы Jetpack Compose UI, включая главный экран (`MainScreen.kt`), пользовательские UI-компоненты (например, `RotatingVinyl.kt`), логику навигации и стилизацию.

### `com.example.player`
Управляет состоянием воспроизведения, медиа-сессиями и `MusicViewModel`, который связывает UI с медиа-контроллером.

### `com.example.data`
Содержит уровень данных, включая базу данных Room (`AppDatabase`), объекты доступа к данным (`TrackDao`) и `TrackRepository`, отвечающий за сложные операции сканирования файлов.

## Как это работает

1. **Выбор папки**
   Пользователь выбирает на устройстве каталог, содержащий его аудиофайлы (например, `.mp3`).

   2. **Умное кеширование**
      Приложение рекурсивно сканирует каталог, извлекая стандартные метаданные (Название, Исполнитель, Обложка, Длительность). Для оптимизации производительности во время жизненного цикла приложения (например, при `onResume`) оно сравнивает характеристики файлов с кешем в локальной базе Room. Только новые или недавно изменённые файлы активно парсятся, что экономит заряд батареи и время.

      3. **Воспроизведение и навигация**
         Выбранные треки воспроизводятся, а UI динамически реагирует на изменения состояния, обновления обложек и события прогресса.
